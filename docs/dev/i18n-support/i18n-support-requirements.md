# 需求文档: i18n Support for GUI Launcher

> **功能标识**: i18n-support
> **复杂度**: standard
> **生成方式**: clarify
> **生成时间**: 2026-01-21

## 1. 概述

### 1.1 一句话描述
为 Windows GUI 启动器添加多语言支持，首期支持英文和简体中文，通过 Qt 原生机制实现。

### 1.2 核心价值
让非英语用户（特别是中文用户）能更轻松地配置和使用 docx-mcp-server，降低使用门槛。

### 1.3 目标用户
- **主要用户**：使用 Windows GUI 启动器的中文用户。

---

## 2. 需求与用户故事

### 2.1 需求清单

| ID | 需求点 | 优先级 | 用户故事 |
|----|--------|--------|----------|
| R-001 | 语言切换 UI | P0 | 作为用户，我希望在界面上能通过下拉框选择语言，以便切换到我熟悉的语言。 |
| R-002 | 中文翻译覆盖 | P0 | 作为中文用户，我希望所有界面文本（按钮、标签、提示）都显示为中文。 |
| R-003 | 设置持久化 | P1 | 作为用户，我希望重启软件后能记住我上次选择的语言。 |
| R-004 | 自动检测 | P2 | 作为新用户，我希望首次启动时自动匹配我的系统语言。 |

### 2.2 验收标准

#### R-001: 语言切换 UI
- **WHEN** 用户在下拉框选择 "简体中文", **THEN** 界面文本立即更新为中文。

#### R-003: 设置持久化
- **WHEN** 用户切换语言并关闭程序, **THEN** 下次启动时应用该语言。

---

## 4. 技术约束

### 4.1 技术栈
- **Framework**: PyQt6
- **Tool**: `lupdate`, `lrelease` (Qt Linguist tools)
- **Format**: `.ts` (XML source), `.qm` (Binary compiled)

### 4.2 打包约束
- `.qm` 文件必须通过 PyInstaller 打包进 EXE（使用 `sys._MEIPASS` 或 Qt Resource System）。
